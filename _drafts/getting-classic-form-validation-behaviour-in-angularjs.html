When I think of classic validation behaviour, I think:

  • The submit handler won't be invoked if the form is invalid
  • Validation messages don't appear initially for empty input fields
  • Validation messages should appear after a form submission, even if the user never modified that field

Out-of-the-box validation

Here's a basic form:

[sourcecode language="html"]
<form name="registration" ng-submit="register()">
<label>
<span>First name:</span>
<input type="text" name="firstName" ng-model="user.firstName" required />
<span class="error" ng-show="registration.firstName.$errors.required">This field is required</span>
</label>
<label>
<span>First name:</span>
<input type="text" name="firstName" ng-model="user.firstName" />
<span class="error" ng-show="registration.firstName.$errors.required">This field is required</span>
</label>
<input type="submit" value="Submit" />
</form> [/sourcecode]
