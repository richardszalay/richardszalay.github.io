Super secret passwords

To avoid hard coding our deployment password, we're going to utilise a feature supported by MSDeploy and msdeploy-package-publish (though not currently by the default Web Publishing Pipeline): the Windows Credential Manager.

You'd be excused for not knowing about the Windows Credential Manager since it's hardly a high profile of Windows. It's basically a per-user encrypted password database, which means only the build server user can access the password.

To register the credentials on your build server, enter the following at the command prompt ("run as" the build server user):

{code}
cmdkey /generic:deploy_user /user:SERVER\deploy_user /password:S3cure!
{code}
